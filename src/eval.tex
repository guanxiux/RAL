\section{Evaluation}
\label{sec:eval}

\subsection{Experimental Setup}
\label{sec:eval:setup}

We evaluate \sysname on four GPU platforms, three robotic perception tasks
with distinct mask sources, and three baselines representing the dense,
tile-skipping, and gather-scatter paradigms.

\subsubsection{Platforms}
Experiments run on two embedded platforms representative of onboard robotic
deployment and two desktop-class GPUs that test generality:
Jetson Xavier NX (8\,GB, CUDA~11.4),
Jetson AGX Orin (64\,GB, CUDA~11.4),
RTX~4070 Laptop (CUDA~12.1), and
RTX~3080 Desktop (CUDA~12.1).

\textbf{Optical flow.} We use EVFlowNet~\cite{evflownet}, a UNet
encoder-decoder, with EvConv~\cite{evconv} as the mask source. We follow
the standard EvConv evaluation protocol on the MVSEC
dataset~\cite{mvsec} ($346{\times}260$ resolution).

\textbf{Object detection.} We use YOLOv11~\cite{yolov11}
with temporal frame differences as the mask source
(DeltaCNN~\cite{deltacnn}): consecutive frames are differenced and
positions exceeding a threshold % TODO: threshold value
are marked active. We evaluate on the MOT16 dataset~\cite{mot16} with
$640{\times}640$ input resolution.

\textbf{Depth estimation.} We use a ResNet-34 encoder-decoder~\cite{monodepth2}
with DGNet~\cite{dgnet} learned spatial gating as the mask source: a
lightweight gating module predicts task-relevant regions per frame.
We evaluate on KITTI~\cite{kitti} ($1280{\times}384$ resolution).

EVFlowNet~\cite{evflownet} and YOLOv11 % TODO: cite yolov11
use their respective official pretrained weights;
EvConv~\cite{evconv} and DeltaCNN~\cite{deltacnn} are inference-time mask
sources that do not modify model weights. For DGNet~\cite{dgnet}, the
gating module is jointly trained with the backbone following the default
DGNet configuration.

\subsubsection{Baselines}

We compare against three baselines, all running in FP32 on identical
hardware and inputs:
\begin{itemize}
  \item \textbf{Dense}: cuDNN~\cite{cudnn} implicit GEMM with best-algorithm selection
        via \texttt{cudnnFind}.
  \item \textbf{Tile skipping}: DeltaCNN~\cite{deltacnn} execution engine.
        DeltaCNN includes a delta-propagation optimization specific to
        temporal frame differences (computing and propagating only output
        deltas). We enable delta propagation for the temporal frame difference task
        and disable it for the event-camera and learned-gating tasks since they are not applicable.
  \item \textbf{Gather-scatter}: MMCV~\cite{mmcv} \texttt{MaskedConv2d}
        operator, which performs \texttt{masked\_im2col} (gather active
        patches), matrix multiply, and \texttt{masked\_col2im} (scatter
        back).
\end{itemize}

\subsubsection{Metrics and Methodology}

We report four metrics:
(1)~end-to-end latency (ms),
(2)~useful-compute ratio $\eta$ (Section~\ref{sec:bg:gap}),
(3)~throughput (GFLOPS, giga floating-point operations per second), and
(4)~task accuracy: Average Endpoint Error (AEE, pixels) for optical flow,
mean Average Precision (mAP, \%) for object detection, and Root Mean
Square Error (RMSE, meters) for depth estimation.
All latency measurements use CUDA events with device synchronization,
averaged over 3 repetitions after 3 warm-up runs. We report standard
deviation.
