\section{\sysname Design}
\label{sec:design}

\subsection{Overview}
\label{sec:design:overview}

Onboard robotic perception demands real-time convolution under strict
compute and power budgets; on embedded platforms, dense convolution often
cannot meet the hard latency ceiling imposed by the control loop
(Section~\ref{sec:bg:dense}). Masked convolution exploits spatial sparsity
to reduce convolution latency by computing only the outputs affected by
active input regions. However, as shown in Section~\ref{sec:bg:gap},
existing approaches fail to convert high sparsity into proportional latency
reduction due to the tension between useful-compute ratio and throughput.
\sysname resolves this tension through a two-stage design
(Figure~\ref{fig:overview}):

\begin{enumerate}
  \item \textbf{Construction stage.} A lightweight kernel scans the input
        active mask $M_{in}$ and evaluates Eq.~\eqref{eq:mask} for each
        output position, emitting an ordered \emph{worklist}
        $\mathbf{q} = (q_1, \ldots, q_m)$ of $m = \|M\|_0$ active output
        indices drawn from $G_{out}$. The worklist preserves tile-local
        spatial locality (Section~\ref{sec:design:construct}).

  \item \textbf{Compute stage.} A convolution kernel consumes $\mathbf{q}$
        as its coordinate source. For each $q_j$, the kernel decodes the
        spatial coordinate, reads the receptive-field input
        $\mathbf{x}[\mathcal{N}(q_j)]$, and accumulates
        Eq.~\eqref{eq:conv} identically to the dense pipeline
        (Section~\ref{sec:design:compute}).
\end{enumerate}

\sysname achieves high useful-compute ratio because the computed set
$\mathcal{C} = \{q_1, \ldots, q_m\}$ equals the set of active positions
(no wasted work). It achieves near-dense throughput through three
mechanisms: the construction stage operates entirely in mask space with
minimal overhead; the worklist preserves locality comparable to the dense
traversal order; and the compute stage's per-position datapath is identical
to the dense pipeline.

\subsection{Worklist Construction}
\label{sec:design:construct}

The construction stage produces the worklist $\mathbf{q}$ from the input
mask $M_{in}$ by performing a per-tile gather followed by concatenation.
It operates on the same tile partition
$\{T_{i,j}\}_{i \in [N_h],\, j \in [N_w]}$ defined in
Eq.~\eqref{eq:tile}.

\paragraph{Output mask evaluation.}
For each linear index $g \in G_{out}$, the construction stage evaluates
Eq.~\eqref{eq:mask} by testing whether the receptive field of $g$
intersects the input mask:
\begin{equation}
  M[g] = \begin{cases}
    1 & \text{if } \exists\, q \in \mathcal{N}(g) \text{ s.t.\ }
        M_{in}[q] = 1, \\
    0 & \text{otherwise,}
  \end{cases}
  \label{eq:mask-eval}
\end{equation}
requiring $k^2$ boolean reads of $M_{in}$ per position.

\paragraph{Per-tile gather.}
For each tile $T_{i,j}$, the output mask $M$ induces a support set of
active indices within the tile:
\begin{equation}
  \mathcal{I}_{i,j} = \bigl\{\, k \in [t_h t_w] : M[T_{i,j}[k]] = 1 \,\bigr\}.
  \label{eq:support}
\end{equation}
The per-tile subsequence is the gather of $T_{i,j}$ by its support set:
\begin{equation}
  \mathbf{q}_{i,j} = (T_{i,j})_{\mathcal{I}_{i,j}}
                    = (T_{i,j}[k])_{k \in \mathcal{I}_{i,j}},
  \label{eq:pertile-gather}
\end{equation}
ordered by ascending $k$ (i.e., row-major within the tile), with length
$|\mathbf{q}_{i,j}| = |\mathcal{I}_{i,j}|$.

\paragraph{Worklist concatenation.}
The $N = N_h \cdot N_w$ tiles are enumerated in row-major order as
$1, \ldots, N$. Let $P = (P_1, \ldots, P_N)$ be the permutation of
$[N]$ determined by nondeterministic parallel block scheduling. The
worklist is:
\begin{equation}
  \mathbf{q} = (\mathbf{q}_{P_1},\; \mathbf{q}_{P_2},\; \ldots,\;
               \mathbf{q}_{P_N}),
  \label{eq:worklist}
\end{equation}
with total length $|\mathbf{q}| = \|M\|_0$. Each $\mathbf{q}_{P_n}$
retains its internal row-major ordering; only the inter-tile order varies
with $P$.


\paragraph{Locality analysis.}
The compute stage processes $\mathbf{q}$ in chunks of $B_M$ consecutive
entries. Consider any contiguous chunk $\mathbf{q}[b:b{+}B_M]$. Because
each per-tile subsequence $\mathbf{q}_{P_n}$ contains at most $t_h t_w$
entries, a chunk spans at most
$\lceil B_M / \min_n |\mathbf{q}_{P_n}| \rceil$ tiles regardless of
the permutation $P$. Within each tile, consecutive entries in
$\mathbf{q}_{P_n}$ are row-major neighbors in $G_{out}$, so their
receptive fields overlap in contiguous memory. The nondeterministic
inter-tile ordering does not degrade locality relative to a hypothetical
identity-permutation (dense raster) ordering, because consecutive tiles
under any permutation access the same set of tile-sized spatial regions.

\paragraph{Cost analysis.}
Mask evaluation (Eq.~\ref{eq:mask-eval}) performs $k^2$ boolean reads per
output position, totaling $O(H_{out} W_{out} k^2)$ work. The gather and
concatenation write $\|M\|_0$ integer indices. The entire stage operates
in 2D mask space: its cost depends on spatial resolution and kernel size
but is independent of $C_{in}$ and $C_{out}$. Because the compute stage
scales with $\|M\|_0 \cdot k^2 \cdot C_{in} \cdot C_{out}$, the
construction overhead becomes negligible as channel dimensions grow
(measured at XX\% in Section~\ref{sec:eval}). The stage also produces the
output mask $M$, which serves as $M_{in}$ for the next layer.

\begin{algorithm}[t]
\caption{Worklist Construction}
\label{alg:construct}
\begin{algorithmic}[1]
\REQUIRE Tiles $\{T_{i,j}\}$, input mask $M_{in}$
\ENSURE Worklist $\mathbf{q}$, output mask $M$
\FOR{each tile $T_{i,j}$ \textbf{in parallel}}
  \FOR{each $k \in [t_h t_w]$}
    \STATE $g \leftarrow T_{i,j}[k]$
    \STATE $M[g] \leftarrow$ Eq.~\eqref{eq:mask-eval}
  \ENDFOR
  \STATE $\mathcal{I}_{i,j} \leftarrow \{k : M[T_{i,j}[k]] = 1\}$
  \STATE $\mathbf{q}_{i,j} \leftarrow (T_{i,j}[k])_{k \in \mathcal{I}_{i,j}}$
\ENDFOR
\STATE $\mathbf{q} \leftarrow (\mathbf{q}_{P_1}, \ldots, \mathbf{q}_{P_N})$
  \hfill \COMMENT{$P$ from block scheduling}
\end{algorithmic}
\end{algorithm}


\subsection{Worklist-driven Convolution}
\label{sec:design:compute}

The compute stage evaluates Eq.~\eqref{eq:conv} for exactly the positions
in $\mathbf{q}$, formulated as an implicit GEMM whose M-dimension is
indexed by the worklist rather than a full raster scan of $G_{out}$.

\paragraph{GEMM formulation.}
Let $m = |\mathbf{q}| = \|M\|_0$. The masked convolution computes:
\begin{equation}
  \mathbf{Y}_{m \times C_{out}} = \mathbf{A}_{m \times k^2 C_{in}}
  \;\cdot\; \hat{\mathbf{W}}_{k^2 C_{in} \times C_{out}},
  \label{eq:gemm}
\end{equation}
where $\hat{\mathbf{W}}$ is the weight tensor reshaped into a 2D matrix,
and row $j$ of $\mathbf{A}$ is the flattened receptive-field input for
worklist entry $q_j$:
\begin{equation}
  \mathbf{A}[j, :] = \bigl[\,
    \mathbf{x}[\mathcal{N}_1(q_j)]^\top,\;
    \ldots,\;
    \mathbf{x}[\mathcal{N}_{k^2}(q_j)]^\top
  \,\bigr].
  \label{eq:implicit-a}
\end{equation}
The matrix $\mathbf{A}$ is never materialized; each element is computed on
the fly by decoding the spatial coordinate of $q_j$ and reading the
corresponding input position from $\mathbf{x}$.

\paragraph{Coordinate decoding.}
Given a worklist entry $q_j$ (a linear index in $G_{out}$), the kernel
recovers the 2D spatial position:
\begin{equation}
  h = \lfloor q_j / W_{out} \rfloor, \quad
  w = q_j \bmod W_{out}.
  \label{eq:decode}
\end{equation}
The receptive-field addresses $\mathcal{N}(q_j)$ follow from $(h, w)$ and
the layer's stride and padding parameters, identically to the dense
pipeline.

\paragraph{Tiled execution.}
The GEMM is partitioned into tiles of size $B_M \times B_N$ (worklist
rows $\times$ output channels), identical to standard implicit GEMM. Each
tile $(b_m, b_n)$ computes the output for $B_M$ consecutive worklist
entries and $B_N$ output channels. The inner loop reduces over the
K-dimension ($k^2 C_{in}$) in steps of $B_K$: at each step, the kernel
decodes the worklist entries $q_j$ via Eq.~\eqref{eq:decode} to gather
the corresponding input block from $\mathbf{x}$, loads the matching weight
block from $\hat{\mathbf{W}}$, and accumulates the partial product. After
all K-steps complete, the result is written back to $\mathbf{y}[q_j]$
using worklist entries as output addresses.
Algorithm~\ref{alg:compute} summarizes this procedure. The only difference
from the dense implicit GEMM is the coordinate source: the worklist
$\mathbf{q}$ replaces the sequential scan of $G_{out}$.

\paragraph{Equivalence to dense execution.}
For any single position $q_j$, the operations (decode coordinate, read
$k^2 \times C_{in}$ input values, multiply-accumulate against
$\mathbf{W}$, write $C_{out}$ outputs) are identical to what the dense
pipeline performs for that position. \sysname changes only \emph{which}
positions are processed ($\mathbf{q}$ instead of the full $G_{out}$) and
\emph{how many} ($\|M\|_0$ instead of $H_{out} W_{out}$). The arithmetic
intensity, memory access pattern per position, and GPU occupancy model are
preserved.

\begin{algorithm}[t]
\caption{Worklist-driven Convolution}
\label{alg:compute}
\begin{algorithmic}[1]
\REQUIRE Worklist $\mathbf{q}$, input $\mathbf{x}$, weight $\hat{\mathbf{W}}$
\ENSURE Output $\mathbf{y}$ at positions in $\mathbf{q}$
\FOR{each tile $(b_m, b_n)$ \textbf{in parallel}}
  \STATE Load worklist segment $\mathbf{q}[b_m]$
  \STATE Initialize accumulator $\mathbf{Y} \leftarrow \mathbf{0}_{B_M \times B_N}$
  \FOR{$k = 1, \ldots, \lceil k^2 C_{in} / B_K \rceil$}
    \STATE $\mathbf{X} \leftarrow$ gather input block from $\mathbf{x}$ at positions $\mathcal{N}(\mathbf{q}[b_m])$, channel tile $k$
    \STATE $\hat{\mathbf{W}}_k \leftarrow$ load weight block for channel tile $k$
    \STATE $\mathbf{Y} \leftarrow \mathbf{Y} + \mathbf{X} \cdot \hat{\mathbf{W}}_k$
  \ENDFOR
  \STATE Write $\mathbf{Y}$ to $\mathbf{y}$ at addresses $\mathbf{q}[b_m]$
\ENDFOR
\end{algorithmic}
\end{algorithm}

\subsection{Multi-layer Integration}
\label{sec:design:integration}

In a multi-layer CNN, \sysname's construction stage produces both the
worklist $\mathbf{q}^l$ and the output active mask $M^l$ for each layer
$l$. The output mask directly serves as $M_{in}^{l+1}$ for the next
layer's construction stage, propagating sparsity through the network
without additional mask-computation passes. \sysname is a drop-in
replacement for standard convolution layers: it accepts the same weight
tensors and produces numerically identical outputs at active positions,
requiring no model retraining or architectural changes.
